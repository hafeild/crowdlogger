Using a CrowdLogger Remote Module

Which CrowdLogger

If you want to program a CLRM for use with a large network of users via, e.g., the UMass CrowdLogger Project or one you are hosting, then you can publish your module. This basically means your module gets placed on a server and is loaded from there. Note that publishing within the UMass CrowdLogger Project requires having your module inspected in much the same way as an app submitted to the Apple App Store or Google Market Place would be.

If your developing or only need a module for a local clone of CrowdLogger, then you can also use the CLRM development or local mode. Your modules will look exactly the same, but they are stored locally on your computer rather than a server.

Development/Local Mode

When just developing or using CrowdLogger in a lab setting, we don't want to worry about setting up a server—even a local one. All you need to do to use a CLRM in local or development mode is to set the CLRM_LOCAL_DIR to the path of the directory containing your CLRM—that's it! CrowdLogger will make all CLRMs in that directory available to the user. 

{
    "CLRM_LOCAL_DIR": "/path/to/dir"
}

Published Mode

To make your CLRM available to a large user base and avoid the need to constantly push out extension updates, you can use the published mode. This involves placing your CLRM's directory in the web/remote-modules/ directory in the web server.

If you are hosting your own CrowdLogger server, this is easy. However, if you want to publish your CLRM in the UMass CrowdLogger Network, then you will need to have it inspected to ensure that your CLRM meets our security and privacy standards. See the CrowdLogger website for more information.
