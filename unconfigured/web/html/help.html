Pausing the logger

By default, the logger will start logging as soon as you install the extension. To pause the logger, click the "pause" button () on the navigation bar. When pressed, a menu will pop up. The first entry is another pause button. Click that to pause logging; the icon will change into a "play" button (). You can press this button to start logging again.

Register

In order for us to understand the demographics of the study participants, and to enter you into the periodic Amazon.com gift card raffle, we ask that you register. If you would like to opt out of (or back into) the raffle, this is where you would do so. This is also where you can enter the code given to you by whoever referred you to %%PROJECT_NAME%%.

While you must enter something for (almost) all of the fields, you can select "I prefer not to answer" for multiple-choice question or enter "PNA" (prefer not to answer) in text fields.

When you click "Register" at the bottom (this will only be click-able once you've responded to all of the questions), the extension will attempt to contact the registration server. If there is a problem, a pop up box will notify you of such. If successful, a prompt will appear, asking if you would like to refer a friend.

Refer a friend

If you click on the "Refer a friend" button in either the menu or the post-registration prompt described above, you will see a dialog that supplies you with the code you need to give to the participants you refer to this study. It will also give you sample text for an email message to send to your friends. (The %%PROJECT_NAME%% team's experience is that spam filters are less likely to be triggered if you send the message yourself.) Your friends should enter this code during their registration. Participants will be given a greater chance of winning the periodic Amazon.com gift card raffles as they refer more participants.

Settings

To access the settings for the extension, select the "Settings" option. One of the settings is your pass phrase. A pass phrase is used to ensure that, if you use %%PROJECT_NAME%% on multiple computers, we encrypt the information we extract from all of your logs the same way. This way, information extracted from your various computers will not appear as though it came from different users, ensuring that we do not falsely decrypt your information. It is important that you use the same pass phrase across all computers.

Another setting concerns experiments. Currently the only option is whether or not to run experiments automatically when they become available. If you select to have experiments run automatically, experiments will run as soon as they are available without your explicit interaction.

Status Page

Clicking the "Status Page" menu item will bring you to the %%PROJECT_NAME%% status page. This page includes several important notifications and tools:
  • Current notifications, such as:
    • that there are pending experiments to run
    • new messages from the study researchers
    • a new version of the consent form
    • a reminder to set your pass phrase, register, and refer friends
    • that you have won a raffle
    • that a new version of the extension is available
  • New messages
  • A list of raffles you have won
  • Tools, such as:
    • a list of the searchers you have conducted
    • removing your local search log from you computer

Notification of winning a raffle

If you win a raffle, a notification will appear in the status page. You will also be notified via a desktop notification. The status page will provide you with the information you need to claim and download your Amazon.com gift card.

View search log

You can view your search history—a list of the searches you have submitted to search engines, as well as their frequency— by selecting the "Search histogram" link under the "Tools" section of the status page. A dialog will pop up listing your searches. You can sort them alphabetically or by frequency.
